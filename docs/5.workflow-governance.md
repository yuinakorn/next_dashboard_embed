# Workflow Governance

เอกสารนี้อธิบาย workflow สำหรับการส่งตรวจ, อนุมัติ, ปฏิเสธ, เผยแพร่, archive และ audit log ของ Dashboard Hub

## Current Phase

ตอนนี้ implementation อยู่ใน **Phase 3 - Governance Workflow** แบบ mock

route ที่เกี่ยวข้อง:

```text
/review
```

ไฟล์ที่เกี่ยวข้อง:

```text
src/app/review/page.tsx
src/app/review/review-queue.tsx
src/lib/portal-types.ts
```

## Status Flow

สถานะหลัก:

- `draft` - dashboard ยังไม่เผยแพร่
- `in_review` - dashboard ถูกส่งเข้าคิวตรวจ
- `published` - dashboard ผ่านการอนุมัติและเผยแพร่แล้ว
- `rejected` - dashboard ถูกตีกลับพร้อมเหตุผล
- `archived` - dashboard เลิกใช้งาน แต่ยังเก็บประวัติ

flow ขั้นต้น:

```text
draft -> in_review -> published
                  -> rejected
published -> archived
```

## Review Queue

หน้า `/review` แสดง dashboard ที่อยู่ในสถานะ `in_review`

ผู้มีสิทธิ์ publish สามารถ:

- preview dashboard
- เขียน review note
- reject
- approve and publish

ใน mock ปัจจุบัน action จะ update state ใน client เท่านั้น ยังไม่ persist ลง database

## Permission Rule

การ approve/publish ใช้ helper:

```text
canPublishDashboard(user, dashboard)
```

กติกาปัจจุบัน:

- `system_admin` publish ได้ทั้งหมด
- `category_admin` publish dashboard ใน category scope ของตนเอง
- role อื่นไม่มี publish permission โดย default

## Audit Events

ตอนนี้มี type:

```ts
type AuditEvent = {
  id: string;
  actorUserId: string;
  actorName: string;
  action: string;
  entityType: "dashboard" | "category" | "permission";
  entityId: string;
  entityTitle: string;
  note: string;
  createdAt: string;
};
```

mock audit events ถูกเก็บใน client state ของ `ReviewQueue`

action ที่เริ่มรองรับ:

- `dashboard.publish`
- `dashboard.reject`
