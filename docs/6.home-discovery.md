# Home Discovery

เอกสารนี้อธิบายแนวทางหน้า Home ของ Dashboard Hub ใน **Phase 4 - Home And Discovery**

## Current Phase

ตอนนี้หน้า `/` ถูกปรับจากหน้าแสดงรายการทั่วไปเป็น operational home สำหรับผู้ใช้ที่ login แล้ว

เป้าหมาย:

- ให้ผู้ใช้เห็น dashboard สำคัญทันที
- ให้ admin/manager เห็นงาน governance ที่ต้องทำ
- ชี้ให้เห็น dashboard ที่มี embed risk
- ทำให้เข้าถึง Catalog, Review Queue, Audit Log และ Create Dashboard ได้เร็ว

## Home Signals

หน้า Home ใช้ signals หลัก:

- `pinnedDashboards`
- `pendingReviewDashboards`
- `recentlyPublishedDashboards`
- `externalOnlyDashboards`
- `popularDashboards`
- `myTeamDashboards`
- `favoriteDashboards`

signals เหล่านี้อยู่ใน:

```text
src/lib/mock-portal-data.ts
```

## Why This Matters

หน้า Home ไม่ควรเป็นแค่รายการ dashboard ล่าสุด เพราะระบบนี้เป็น governed portal

ผู้ใช้ควรเห็น:

- อะไรสำคัญ
- อะไรต้องตรวจ
- อะไรเผยแพร่ใหม่
- อะไรเปิด iframe ไม่ได้
- อะไรเกี่ยวกับทีมของตัวเอง
