# Embed Provider Strategy

เอกสารนี้กำหนดแนวทางรองรับ dashboard/embed จากระบบภายนอก และ policy สำหรับ URL ที่ iframe ไม่ได้

## Core Rule

การที่เว็บปลายทางสร้างด้วย Next.js, React, Vue หรือ framework ใด ไม่ได้แปลว่าจะฝังใน iframe ได้

สิ่งที่ตัดสินว่า iframe ได้หรือไม่ได้คือ security policy ของเว็บปลายทาง เช่น:

- `X-Frame-Options`
- `Content-Security-Policy frame-ancestors`
- Cloudflare challenge / bot protection
- authentication/session policy
- cookie `SameSite`

ดังนั้นระบบต้องมี fallback เป็น external link เสมอ

## Embed Status

ระบบใช้สถานะ embed เบื้องต้น:

- `embeddable` - คาดว่า iframe ได้ เช่น Looker Studio embed URL
- `unknown` - ต้องทดสอบหรือขึ้นกับ provider config
- `external_only` - ควรเปิดเป็น external link เป็นหลัก
- `blocked` - URL ไม่ถูกต้องหรือรู้ว่าฝังไม่ได้

type อยู่ที่:

```text
src/lib/portal-types.ts
```

logic ประเมินเบื้องต้นอยู่ที่:

```text
src/lib/embed-policy.ts
```

## Looker Studio

ควรใช้ URL ที่มีรูปแบบ:

```text
https://datastudio.google.com/embed/reporting/{reportId}/page/{pageId}
https://lookerstudio.google.com/embed/reporting/{reportId}/page/{pageId}
```

ถ้าเป็นลิงก์ report ปกติที่ไม่มี `/embed/reporting/` ควรแจ้งเตือนให้ผู้ใช้เอา embed URL มาแทน

## Superset

Superset iframe support ขึ้นกับการตั้งค่า:

- embedded dashboard feature
- guest token
- allowed domains
- CSP headers
- auth/session model

สถานะเริ่มต้นควรเป็น `unknown` จนกว่าจะทดสอบกับ instance จริง

## Power BI

Power BI อาจต้องใช้:

- public embed
- secure embed
- embed token
- Microsoft auth

สถานะเริ่มต้นควรเป็น `unknown` และต้องมี external fallback

## Normal Website URL

URL เว็บทั่วไป เช่น:

```text
https://telemed.chiangmaihealth.go.th
```

ควรถือเป็น `external_only` ก่อน เพราะเว็บทั่วไปมักตั้ง header ป้องกัน iframe เช่น:

```text
X-Frame-Options: SAMEORIGIN
Content-Security-Policy: frame-ancestors 'self'
```

ถ้าต้องการให้ฝังได้ เจ้าของเว็บปลายทางต้องปรับ header ให้ยอมรับ portal domain เช่น:

```text
Content-Security-Policy: frame-ancestors 'self' https://dashboard-portal.example.go.th
```

และต้องไม่ส่ง:

```text
X-Frame-Options: SAMEORIGIN
X-Frame-Options: DENY
```

## Portal Behavior

### Create Form

เมื่อผู้ใช้กรอก embed URL:

- validate ว่าเป็น HTTPS
- ประเมิน provider + URL pattern
- แสดงคำเตือนถ้าเป็น external-only
- แสดง iframe preview เฉพาะ URL ที่ไม่ใช่ external-only

### Viewer Page

หน้า `/dashboards/[id]`:

- ถ้า `embeddable` หรือ `unknown` ให้ลอง iframe
- ถ้า `external_only` หรือ `blocked` ให้แสดงคำอธิบายและปุ่มเปิด external dashboard
- แสดง fallback link เสมอ

### Catalog

หน้า `/catalog` ควรแสดง embed status เพื่อให้ admin เห็นว่า dashboard ไหนฝังได้หรือควรเปิดภายนอก

## Future Server-Side Check

ระบบมี endpoint สำหรับ server-side check แล้ว:

```text
POST /api/embed/check
```

request:

```json
{
  "url": "https://example.com"
}
```

response จะสรุป:

- embed status
- reason/recommendation
- HTTP status
- final URL
- `X-Frame-Options`
- `Content-Security-Policy`
- server header

สิ่งที่ endpoint ตรวจ:

- ยิง `HEAD` หรือ `GET` ไปดู header ปลายทาง
- ตรวจ `X-Frame-Options`
- ตรวจ `Content-Security-Policy frame-ancestors`
- ตรวจ HTTP status เช่น 403/401/5xx
- timeout 8 วินาที

ในอนาคตควรบันทึกผลเป็น `embed_health`

ข้อจำกัด:

- บางเว็บตอบ header ต่างกันตาม user-agent
- บางเว็บมี Cloudflare challenge
- บางเว็บต้อง login ก่อนจึงจะเห็น header จริง
- browser เป็นผู้บังคับใช้ iframe policy ขั้นสุดท้ายอยู่ดี
