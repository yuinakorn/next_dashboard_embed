# Product

## Register

product

## Users

Dashboard Hub is used by health data admins, category owners, project managers, editors, reviewers, and viewers who need a governed way to find, open, publish, review, and audit embedded dashboards. They work in an operational context where speed, permission clarity, and trust in metadata matter more than marketing polish.

## Product Purpose

The portal centralizes dashboards from providers such as Looker Studio, Superset, Grafana, Metabase, Power BI, and custom URLs. It makes dashboard status, owner, sensitivity, embed health, review state, and audit traceability visible so teams can publish safely and users can open the right dashboard quickly.

## Brand Personality

Calm, accountable, operational. The interface should feel like a serious internal command surface: precise enough for governance, warm enough for daily use, and confident without looking decorative.

## Anti-references

Avoid generic SaaS landing-page composition, identical card grids, heavy decorative gradients, glassmorphism, healthcare-cliche teal-on-white styling, and admin screens where every panel has the same visual weight. The UI should not feel like a template dashboard with interchangeable metric cards.

## Design Principles

1. Surface governance state before decoration.
2. Make every panel earn its place in the workflow.
3. Use density when it improves scanning, not when it hides priority.
4. Reserve color for state, risk, focus, and primary action.
5. Keep Thai and English labels readable under realistic operational text lengths.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-visible focus states, reduced-motion-friendly transitions, and semantic HTML. State should not rely on color alone; status labels, counts, and supporting text must remain explicit.
